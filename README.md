# repo-to-skill

Generate agent skills from any GitHub repository. Give it a repo URL, get back a tested, ready-to-use skill.

## What it does

`repo-to-skill` takes an open-source GitHub repo and automatically:

1. **Clones & analyzes** — README, docs, examples, entry points
2. **Classifies** — CLI tool, Python library, framework, or service
3. **Generates** — Complete skill with SKILL.md, references, and scripts
4. **Tests** — Runs installation and smoke tests
5. **Evaluates** — Benchmarks with-skill vs baseline using test prompts

## Install

```bash
npx skills add shuyhere/repo-to-skill
```

Or manually copy the skill folder into your agent's skills directory.

## Usage

### As a slash command (OpenClaw / Claude Code)

```
/skill repo-to-skill https://github.com/vllm-project/vllm
```

### As a prompt

> Create a skill for using https://github.com/hiyouga/LLaMA-Factory

The agent will follow the skill's 6-step process to produce a complete, evaluated skill.

## What you get

```
<tool-name>/
├── SKILL.md                    # Core instructions (under 500 lines)
├── references/
│   ├── api-reference.md        # Full API/CLI reference
│   ├── examples.md             # Curated usage examples
│   └── configuration.md        # Config options
├── scripts/
│   └── setup.sh                # Installation script
└── evals/
    └── evals.json              # Test cases with assertions
```

## Included tools

- **`scripts/analyze_repo.sh`** — Automated repo analyzer (clones, detects language, extracts structure, README, docs, examples, entry points)
- **`references/skill-format.md`** — Agent skill format reference
- **`references/eval-schemas.md`** — Evaluation JSON schemas (evals.json, grading.json, benchmark.json)

## Evaluation framework

Skills generated by repo-to-skill include built-in evaluation:

- **Test prompts** — 3-5 realistic user prompts per skill
- **Assertions** — Verifiable expectations for each prompt
- **With-skill vs baseline** — Side-by-side comparison
- **Grading** — Automated pass/fail with evidence
- **Benchmarking** — Aggregate pass_rate, time, and token usage

## Example

Generated a [llamafactory skill](https://github.com/hiyouga/LLaMA-Factory) that scored **22/22 (100%)** on all eval assertions vs **21/22 (95.5%)** baseline.

## Compatible with

- [OpenClaw](https://github.com/openclaw/openclaw) — via native skill commands
- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) — via `/skill` or prompt
- [Codex](https://openai.com/codex) — via AGENTS.md skill loading
- Any agent that supports the [Agent Skills](https://agentskills.io) format
